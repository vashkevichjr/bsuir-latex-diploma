# BSUIR LaTeX Diploma Workspace

Репозиторий предназначен для подготовки пояснительной записки в `LaTeX` под требования БГУИР. Внутри уже настроены преамбула, структура по разделам, папка для изображений и набор рабочих `docs`, которые можно использовать как источник контекста для ручной работы или генерации текста через ИИ-агента.

## Что лежит в репозитории

- `preamble.tex` — общие правила оформления документа;
- `main.tex` — точка сборки итогового PDF;
- `tex/` — исходники разделов записки;
- `images/` — все изображения, используемые в документе;
- `docs/diploma-topic-context.md` — предметная область, тема, роли, API, ограничения;
- `docs/diploma-generation-instructions.md` — контрольная структура диплома: разделы, таблицы, рисунки;
- `docs/diploma-generation-plan.md` — рабочий план генерации и текущее состояние проекта.

## Требования к окружению

Минимально нужно:

- `TeX Live` с `xelatex`;
- `latexmk`;
- любой редактор, удобный для работы с `LaTeX`.

Опционально:

- `VS Code` с `LaTeX Workshop`;
- `git` для фиксации прогресса;
- ИИ-агент, работающий в этом репозитории и использующий файлы из `docs/` как контекст.

## Как собрать PDF

Из корня проекта:

```bash
latexmk -xelatex -interaction=nonstopmode -halt-on-error main.tex
```

Результат сборки:

- `main.pdf` — итоговая записка;
- `main.log` — лог сборки.

Очистка вспомогательных файлов:

```bash
latexmk -c
```

## Как пользоваться репозиторием для генерации диплома

Рекомендуемый порядок работы:

1. Заполнить `docs/diploma-topic-context.md`.
   Здесь фиксируются тема диплома, предметная область, роли, сущности, стек, бизнес-логика, ограничения и терминология.

2. Актуализировать `docs/diploma-generation-instructions.md`.
   В этом файле хранится контрольная структура документа:
   - порядок разделов и подразделов;
   - перечень обязательных рисунков;
   - перечень обязательных таблиц;
   - правила оформления.

3. Вести статус в `docs/diploma-generation-plan.md`.
   Этот файл нужен как рабочий трекер:
   - что уже написано;
   - какие разделы еще пустые;
   - какие изображения и таблицы уже готовы;
   - что нужно сделать следующим шагом.

4. Складывать изображения в `images/`.
   Все рисунки, на которые ссылается структура, должны лежать именно там и подключаться из `tex/*.tex`.

5. Генерировать или редактировать разделы в `tex/`.
   Обычно удобно держать по одному файлу на раздел:
   - `tex/introduction.tex`
   - `tex/01-analysis.tex`
   - `tex/02-modeling.tex`
   - `tex/03-design.tex`
   - `tex/04-testing.tex`
   - `tex/04-economics.tex`
   - `tex/sources.tex`

6. После каждой крупной правки пересобирать `main.pdf` и проверять:
   - нумерацию разделов;
   - подписи рисунков и таблиц;
   - переносы в таблицах;
   - отсутствие `Overfull` и критических ошибок в `main.log`.

## Как работать с ИИ-агентом

Если репозиторий используется как рабочее пространство для генерации записки через ИИ, агенту нужно давать три источника истины:

- `docs/diploma-topic-context.md` — что за тема и что можно/нельзя писать;
- `docs/diploma-generation-instructions.md` — какая структура и какие артефакты обязательны;
- `docs/diploma-generation-plan.md` — что уже сделано и что делать дальше.

Практический сценарий:

1. Обновить контекст темы.
2. Обновить план.
3. Попросить агента сделать конкретный раздел или подраздел.
4. Передать недостающие изображения в `images/`.
5. Проверить сборку PDF.
6. Зафиксировать результат в `git`.

## Правила оформления, которые уже зашиты в проект

- латиница в основном тексте оформляется через `\eng{...}`;
- ссылки и `URL` оформляются через `\url{...}`;
- заголовок страницы содержания — `Содержание`, по центру;
- таблицы собираются с переносом текста внутри ячеек;
- подписи таблиц и `Продолжение таблицы` выводятся без красной строки;
- документ собирается через `XeLaTeX`.

## Работа с git

Типовой цикл:

```bash
git status
git add <нужные_файлы>
git commit -m "Update diploma sections"
git push origin main
```

Перед коммитом полезно убедиться, что в индекс не попали:

- временные изображения;
- старые черновики;
- промежуточные файлы, не используемые в сборке.

## Примечание

Этот репозиторий не привязан к одному конкретному внешнему шаблону. Основной способ работы — поддерживать структуру, тему и артефакты непосредственно в `docs/` и `tex/`, а итоговый документ собирать напрямую в `LaTeX`.
