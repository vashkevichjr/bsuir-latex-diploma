# BSUIR LaTeX Diploma Workspace

Этот репозиторий можно использовать как готовую основу для пояснительной записки
дипломного проекта в `LaTeX`. Внутри уже есть:

- настроенная преамбула и сборка через `XeLaTeX`;
- разбиение записки по разделам в `tex/`;
- папка `images/` для рисунков;
- папка `docs/` с контекстом темы, правилами генерации и рабочим планом;
- папка `Графическая часть/` для плакатов и итоговой графики.

Репозиторий рассчитан на два сценария:

1. ручная работа над запиской;
2. работа вместе с ИИ-агентом, который использует `docs/*.md` как контекст.

## Структура репозитория

- `main.tex` — точка сборки итогового `PDF`;
- `preamble.tex` — оформление, шрифты, таблицы, макросы;
- `tex/` — разделы записки;
- `images/` — рисунки, используемые внутри записки;
- `Графическая часть/` — плакаты и итоговые графические материалы;
- `docs/diploma-topic-context.md` — контекст темы и предметной области;
- `docs/diploma-generation-instructions.md` — инструкция по генерации и адаптации;
- `docs/diploma-generation-plan.md` — рабочий план и текущий статус.

## Быстрый старт

Из корня проекта:

```bash
latexmk -xelatex -interaction=nonstopmode -halt-on-error main.tex
```

Результат:

- `main.pdf` — итоговая пояснительная записка;
- `main.log` — лог сборки.

Очистка промежуточных файлов:

```bash
latexmk -c
```

Полная очистка, включая `pdf`:

```bash
latexmk -C
```

## Установка окружения

### macOS

Рекомендуемый вариант через `Homebrew`:

```bash
brew install --cask mactex-no-gui
```

После установки перезапусти терминал и проверь:

```bash
xelatex --version
latexmk --version
```

Если `xelatex` не находится в `PATH`, добавь:

```bash
export PATH="/Library/TeX/texbin:$PATH"
```

в `~/.zshrc` или `~/.bashrc`.

### Windows

Рекомендуемый вариант для совместимости:

1. установить `TeX Live` или `MiKTeX`;
2. если используется `MiKTeX`, дополнительно установить `Strawberry Perl`, чтобы работал `latexmk`;
3. открыть `PowerShell` в корне проекта.

Проверка:

```powershell
xelatex --version
latexmk --version
```

Сборка:

```powershell
latexmk -xelatex -interaction=nonstopmode -halt-on-error main.tex
```

Если используется `MiKTeX`, в первый запуск он может доустановить недостающие пакеты.

### Linux

#### Ubuntu / Debian

Самый простой вариант:

```bash
sudo apt update
sudo apt install -y texlive-full latexmk
```

#### Fedora

```bash
sudo dnf install -y texlive-scheme-full latexmk
```

#### Arch Linux

```bash
sudo pacman -S texlive-meta latexmk
```

Проверка:

```bash
xelatex --version
latexmk --version
```

## Как использовать решение под свой диплом

### 1. Обновить тему и предметную область

Открыть `docs/diploma-topic-context.md` и заменить:

- тему;
- предметную область;
- роли пользователей;
- сущности базы данных;
- стек технологий;
- терминологию;
- список запрещенных старых сущностей, если они остались от шаблона.

Это главный файл контекста. Если тема меняется, сначала обновляется он.

### 2. Обновить правила генерации

Открыть `docs/diploma-generation-instructions.md` и зафиксировать:

- нужную структуру записки;
- обязательные разделы и подразделы;
- рисунки;
- таблицы;
- приложения;
- правила оформления.

Если используется другой вузовский шаблон или другая методичка, менять нужно
этот файл, а не только `tex/`.

### 3. Обновить рабочий план

Открыть `docs/diploma-generation-plan.md` и привести в актуальное состояние:

- что уже написано;
- что еще не готово;
- что должно войти в следующую опроцентовку;
- какие плакаты и приложения уже есть.

### 4. Заменить или добавить изображения

Все рисунки для записки хранятся в `images/`.

Рекомендации:

- использовать понятные имена в формате `fig-<section>-<name>.png`;
- не класть в `images/` временный мусор;
- финальные плакаты держать в папке `Графическая часть/`, а не вперемешку с
  иллюстрациями для записки.

### 5. Отредактировать разделы в `tex/`

Обычно используются такие файлы:

- `tex/introduction.tex`
- `tex/01-analysis.tex`
- `tex/02-modeling.tex`
- `tex/03-design.tex`
- `tex/04-testing.tex`
- `tex/04-economics.tex`
- `tex/conclusion.tex`
- `tex/sources.tex`
- `tex/appendices.tex`

Если структура другая, меняются и имена файлов, и `\input{...}` в `main.tex`.

## Как работать с ИИ-агентом

Если репозиторий используется как рабочее пространство для ИИ-агента, ему
нужно явно указать три файла:

- `docs/diploma-topic-context.md`
- `docs/diploma-generation-instructions.md`
- `docs/diploma-generation-plan.md`

Рекомендуемый цикл:

1. обновить контекст темы;
2. обновить инструкцию по структуре;
3. обновить рабочий план;
4. попросить агента сделать конкретный раздел;
5. пересобрать `PDF`;
6. проверить рисунки, таблицы и нумерацию;
7. зафиксировать изменения в `git`.

## Что уже зашито в преамбулу

В текущем шаблоне уже настроены:

- латиница через `\eng{...}`;
- ссылки через `\url{...}`;
- заголовок `Содержание` по центру;
- перенос текста в таблицах;
- подписи таблиц и `Продолжение таблицы` без красной строки;
- сборка через `XeLaTeX`.

Перед переносом шаблона под новый диплом полезно проверить `preamble.tex`,
чтобы понять, какие правила уже реализованы на уровне макросов.

## Проверка после правок

После каждого крупного изменения стоит проверять:

- собирается ли `main.pdf`;
- нет ли критических ошибок в `main.log`;
- не съехали ли таблицы;
- не пропали ли ссылки на рисунки;
- корректно ли обновилось оглавление;
- не остались ли фрагменты старой предметной области.

Для быстрой сборки:

```bash
latexmk -xelatex -interaction=nonstopmode -halt-on-error main.tex
```

Для поиска явных проблем в логе:

```bash
rg "Overfull|LaTeX Error|Undefined control sequence" main.log
```

## Работа с графической частью

Финальные плакаты и связанные с ними файлы лучше хранить в папке
`Графическая часть/`.

Практика, которая здесь уже используется:

- в корне `Графическая часть/` лежат только финальные версии;
- черновики и вспомогательные экспортные файлы можно убирать в подпапку
  `draft/`.

Это позволяет не смешивать плакаты, изображения из записки и случайные
экспортные файлы.

## Работа с git

Типовой цикл:

```bash
git status
git add README.md docs/*.md tex/*.tex main.tex
git commit -m "Update diploma docs and build instructions"
git push origin main
```

Перед коммитом проверь, что в индекс не попали:

- временные `PDF` и `PNG`;
- старые архивы;
- промежуточные служебные файлы;
- чужие черновики, не относящиеся к задаче.

## Практический минимум для нового пользователя

Если кто-то из друзей просто хочет взять это решение и адаптировать под свой
диплом, минимальный набор шагов такой:

1. установить `TeX Live` / `MacTeX` / `MiKTeX`;
2. собрать `main.pdf`;
3. изучить `docs/diploma-topic-context.md`;
4. заменить тему и предметную область;
5. заменить рисунки и таблицы под свою работу;
6. обновить `tex/`-разделы;
7. пересобрать документ и проверить результат.

Этого достаточно, чтобы использовать репозиторий как рабочую основу, а не как
жестко зашитый диплом под одну конкретную тему.
