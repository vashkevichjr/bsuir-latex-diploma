# Инструкция по генерации и адаптации дипломной записки

Этот файл нужен не как описание одной конкретной темы, а как рабочая инструкция
для использования репозитория под новый диплом.

Если вы берете это решение как основу, логика работы такая:

1. сначала обновляется `docs/diploma-topic-context.md`;
2. затем актуализируется этот файл;
3. затем обновляется `docs/diploma-generation-plan.md`;
4. только после этого редактируются `tex/*.tex`.

## Что считать источниками истины

- `docs/diploma-topic-context.md` — смысл диплома, тема, стек, роли, ограничения;
- `docs/diploma-generation-instructions.md` — структура, правила и порядок генерации;
- `docs/diploma-generation-plan.md` — фактический статус проекта и очередь работ.

Если между файлами есть противоречие:

1. структура и оформление берутся из этого файла;
2. тема и предметная область — из `diploma-topic-context.md`;
3. текущий статус — из `diploma-generation-plan.md`.

## Базовая структура записки

В типовом случае используются следующие блоки:

- Титульный лист;
- Реферат;
- Задание;
- Содержание;
- Введение;
- Раздел 1. Аналитическая часть;
- Раздел 2. Моделирование и требования;
- Раздел 3. Проектирование и разработка;
- Раздел 4. Тестирование и документация;
- Раздел 5. Экономическое обоснование;
- Заключение;
- Список использованных источников;
- Приложения.

Если методичка вуза требует другую структуру, править нужно именно этот список
и соответствующие `\input{...}` в `main.tex`.

## Рекомендуемая файловая структура

### Основные файлы

- `main.tex` — точка сборки;
- `preamble.tex` — оформление и макросы;
- `tex/` — текст разделов;
- `images/` — рисунки для записки;
- `Графическая часть/` — плакаты и отдельные графические материалы;
- `docs/` — контекст, инструкция, план.

### Типовой набор файлов в `tex/`

- `tex/introduction.tex`
- `tex/01-analysis.tex`
- `tex/02-modeling.tex`
- `tex/03-design.tex`
- `tex/04-testing.tex`
- `tex/04-economics.tex`
- `tex/conclusion.tex`
- `tex/sources.tex`
- `tex/appendices.tex`

## Как адаптировать репозиторий под новый диплом

### Шаг 1. Заменить предметную область

В `docs/diploma-topic-context.md` нужно обновить:

- тему;
- предметную область;
- роли;
- ключевые сущности;
- стек;
- пользовательские сценарии;
- ограничения;
- список старых запрещенных шаблонных сущностей.

### Шаг 2. Зафиксировать структуру

В этом файле нужно зафиксировать:

- какие разделы обязательны;
- какие подразделы обязательны;
- какие таблицы должны быть в каждом разделе;
- какие рисунки должны быть в каждом разделе;
- нужны ли плакаты, приложения, листинги и внешние материалы.

### Шаг 3. Зафиксировать план работ

В `docs/diploma-generation-plan.md` необходимо хранить:

- что уже написано;
- что еще пустое;
- какая версия готовится;
- какие замечания уже закрыты;
- что должно войти в следующую сдачу.

## Правила работы с рисунками

Все рисунки для записки кладутся в `images/`.

Рекомендуемый формат имен:

```text
fig-<номер_раздела>-<номер_рисунка>-<краткое-имя>.png
```

Примеры:

- `fig-1-1-dashboard.png`
- `fig-2-4-use-case.png`
- `fig-3-16-er-diagram.png`

Правила:

- один рисунок — один файл;
- временные экспортные файлы не должны оставаться в `images/`;
- плакаты не смешиваются с иллюстрациями записки и лежат в `Графическая часть/`.

## Правила работы с таблицами

При подготовке таблиц нужно проверять:

- текст не выходит за границы ячеек;
- используются колонки с переносом (`P{...}`, `L`, `Y`);
- подписи `Таблица` и `Продолжение таблицы` не имеют красной строки;
- нумерация соответствует структуре разделов.

## Правила текста и оформления

В текущем шаблоне уже приняты следующие правила:

- латиница в основном тексте — через `\eng{...}`;
- `URL` и домены — через `\url{...}`;
- заголовок страницы содержания — `Содержание` и по центру;
- документ собирается через `XeLaTeX`;
- таблицы должны корректно переносить длинный текст.

Перед переносом решения под новый диплом нужно проверить, что эти правила
подходят и новой методичке.

## Как генерировать текст через ИИ

Если разделы пишет ИИ-агент, ему нужно дать:

- `docs/diploma-topic-context.md`
- `docs/diploma-generation-instructions.md`
- `docs/diploma-generation-plan.md`

Рабочий цикл:

1. обновить контекст темы;
2. обновить инструкцию;
3. обновить план;
4. поставить агенту задачу на конкретный раздел;
5. пересобрать `PDF`;
6. визуально проверить результат;
7. зафиксировать изменения в `git`.

## Что проверять после каждой правки

- документ собирается без критических ошибок;
- номера разделов, таблиц и рисунков корректны;
- нет смешения старой и новой предметной области;
- рисунки находятся в правильных разделах;
- таблицы не ломают верстку;
- список источников соответствует текущей теме;
- приложения и плакаты не потеряны.

## Минимальный набор команд

Сборка:

```bash
latexmk -xelatex -interaction=nonstopmode -halt-on-error main.tex
```

Очистка:

```bash
latexmk -c
```

Проверка лога:

```bash
rg "Overfull|LaTeX Error|Undefined control sequence" main.log
```

## Что хранить в графической части

В папке `Графическая часть/` рекомендуется держать:

- финальные плакаты;
- `PDF`-версии графических листов;
- при необходимости исходники `drawio` или другие редактируемые форматы.

Черновики лучше убирать в подпапку `draft/`.

## Что не стоит делать

- не править тему только в `tex/*.tex`, оставляя старый контекст в `docs/`;
- не складывать плакаты в `images/`;
- не оставлять в корне репозитория случайные архивы и временные экспорты;
- не смешивать в одном коммите правки по тексту записки и посторонний файловый мусор;
- не использовать репозиторий как «готовый диплом без адаптации» — его нужно
  обязательно пересобрать под свою тему и структуру.
