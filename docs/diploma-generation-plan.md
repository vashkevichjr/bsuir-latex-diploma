# План генерации диплома

## Актуализация структуры под все 3 опроцентовки

Ниже зафиксирована финальная структура пояснительной записки, которую нужно собрать полностью.
Важно: ранее текущий `tex` был фактически собран под 2-ю опроцентовку, поэтому в проекте уже есть
материал по аналитике, требованиям, проектированию и экономике, но еще не приведены к финальному виду
все обязательные блоки полной записки.

Финальная целевая структура:

- Титульный лист.
- Реферат.
- Задание.
- Содержание.
- Введение.
- `4.1` Анализ литературных источников и программных решений по теме дипломного проекта.
- `4.1.1` Описание и анализ предметной области.
- `4.1.2` Обзор функциональности аналогов программного средства.
- `4.2` Моделирование предметной области и разработка требований к программному средству.
- `4.2.1` Анализ и формализация бизнес-процессов предметной области.
- `4.2.2` Разработка спецификации требований к программному средству.
- `4.2.3` Образ предлагаемого решения.
- `4.3` Проектирование и разработка программного средства.
- `4.3.1` Архитектурные решения и технологии реализации программного средства.
- `4.3.2` Проектирование и разработка пользовательского интерфейса.
- `4.3.3` Разработка модели данных.
- `4.3.4` Описание организации бизнес-логики программного средства.
- `4.3.5` Спецификация к разработанному API и механизмы обеспечения информационной безопасности.
- `4.4` Тестирование и программная документация к программному средству.
- `4.4.1` Функциональное тестирование программного средства.
- `4.4.2` Модульное и интеграционное тестирование бизнес-логики программного средства.
- `4.4.3` Руководство по развертыванию программного средства.
- `4.4.4` Руководство пользователя.
- `4.5` Экономическое обоснование.
- Заключение.
- Список использованных источников.
- Приложения: отчет `Антиплагиат`, текст программы, ведомость дипломного проекта.

## Контроль опроцентовок

### 1-я опроцентовка, 40%

Покрывает:

- `4.1`
- `4.2`
- `4.5`
- `5.1`

Статус относительно текущего проекта:

- `4.1` — материал есть.
- `4.2` — материал есть.
- `4.5` — материал есть как текущий экономический раздел.
- `5.1` — в текущем проекте это не отдельный пункт, нужна сверка с методичкой/заданием.

### 2-я опроцентовка, 60%

Покрывает:

- `4.3`
- `5.2`–`5.5`

Статус относительно текущего проекта:

- `4.3` — материал есть.
- Текущая версия проекта фактически и была собрана под этот этап.

### 3-я опроцентовка, 80%

Покрывает:

- Введение.
- Заключение.
- `4.4`
- `5.6`

Статус относительно текущего проекта:

- Введение — есть.
- Заключение — отсутствует.
- `4.4` Тестирование и программная документация — материал собран как полноценный раздел.
- `5.6` — нужна отдельная сверка с формулировкой из задания/методички.

## Критичные расхождения текущей сборки с финальной структурой

- В документе пока нет титульного листа.
- В документе пока нет реферата.
- В документе пока нет задания.
- В документе уже есть полноценный раздел `4`.
- В документе пока нет заключения.
- В документе пока нет приложений.
- В самой записке используется фактическая нумерация разделов `1`–`5`; обозначения `4.1`–`4.5` относятся к структуре задания.

Этот план задает порядок генерации LaTeX-версии пояснительной записки по фактической структуре,
зафиксированной в `docs/diploma-generation-instructions.md`.

Главное правило: структура, разделы, подразделы, таблицы и изображения должны совпадать 1 в 1
с контрольной инструкцией. Нельзя добавлять или удалять структурные элементы без отдельного явного решения.

## Контрольные числа

- Использований изображений: ровно 26.
- Уникальных файлов изображений: ровно 26.
- Логических таблиц: ровно 20.
- Все изображения должны быть отдельными файлами (целевые рисунки отличаются по смыслу).

## Зафиксированные правила оформления (действуют во всем проекте)

- Все латинские слова и обозначения, кроме ссылок и URL, оформляются через `\eng{...}` (курсив).
- Ссылки, домены и URL оформляются через `\url{...}` без курсива.
- Заголовок страницы содержания: `Содержание`, по центру.
- Текст в таблицах переносится внутри ячеек и не выходит за границы (колонки `P{...}`, `L`, `Y`).
- У таблиц/подписей таблиц/`Продолжение таблицы ...` базовый отступ (без красной строки).

## Контроль после генерации каждого раздела

После генерации каждого раздела нужно проверять:

- все латинские слова и обозначения, кроме ссылок и URL, оформлены через `\eng{...}`;
- ссылки и домены оформлены через `\url{...}` без курсива;
- текст в таблицах переносится внутри ячеек и не выходит за рамки;
- у таблиц/подписей таблиц/продолжений таблиц базовый отступ;
- структура раздела совпадает с планом и контрольной инструкцией;
- номера таблиц и рисунков совпадают с контрольной инструкцией;
- рисунки подключаются из `images/` и находятся в правильных разделах.
