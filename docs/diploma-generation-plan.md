# План работы с дипломной запиской

Этот файл нужен как рабочий статус проекта. Он должен показывать не абстрактную
структуру, а текущее состояние репозитория: что уже собрано, что еще не готово
и что нужно сделать следующим шагом.

## Текущее состояние репозитория

На текущий момент в репозитории уже есть:

- рабочая сборка `main.pdf`;
- разбиение текста по разделам в `tex/`;
- контекст темы в `docs/diploma-topic-context.md`;
- инструкция по генерации в `docs/diploma-generation-instructions.md`;
- папка `Графическая часть/` для плакатов и графики.

## Порядок адаптации под новый диплом

### Шаг 1. Контекст

- обновить `docs/diploma-topic-context.md`;
- удалить или переписать старые сущности;
- зафиксировать новый стек и терминологию.

### Шаг 2. Структура

- обновить `docs/diploma-generation-instructions.md`;
- сверить структуру с методичкой;
- проверить обязательные рисунки, таблицы и приложения.

### Шаг 3. Текст

- пройтись по `tex/*.tex`;
- убрать остатки старой темы;
- адаптировать разделы `1`–`5`;
- обновить заключение и источники.

### Шаг 4. Графика

- проверить `images/`;
- обновить плакаты в `Графическая часть/`;
- убрать черновики в `draft/`, если они не нужны в финале.

### Шаг 5. Сборка и проверка

- собрать `main.pdf`;
- проверить `main.log`;
- вручную открыть `PDF` и плакаты;
- только после этого делать коммит.

## Что проверять перед каждой сдачей

### По записке

- документ открывается;
- оглавление корректно;
- нумерация разделов, таблиц и рисунков не съехала;
- нет старой предметной области;
- латиница, ссылки и таблицы оформлены единообразно.

### По плакатам

- все финальные плакаты лежат в `Графическая часть/`;
- названия файлов понятны и соответствуют назначению;
- `PDF`-файлы открываются;
- в папке нет случайного мусора в корне.

### По приложениям

- листинги актуальны;
- внешние материалы не потеряны;
- в записке есть ссылки на нужные приложения.

## Рекомендуемый ежедневный цикл работы

1. проверить `git status`;
2. обновить `docs`, если поменялась тема, структура или план;
3. внести правки в `tex/` или `images/`;
4. собрать `main.pdf`;
5. проверить лог;
6. зафиксировать изменения в `git`.

## Команды

Сборка:

```bash
latexmk -xelatex -interaction=nonstopmode -halt-on-error main.tex
```

Проверка явных ошибок:

```bash
rg "Overfull|LaTeX Error|Undefined control sequence" main.log
```

Просмотр измененных файлов:

```bash
git status --short
```

## Признаки, что репозиторий готов к передаче другому человеку

- `README.md` объясняет установку и сборку;
- `docs/` отражают актуальное состояние;
- плакаты лежат отдельно от картинок записки;
- сборка воспроизводима на новой машине;
- новый пользователь понимает, какие файлы менять под свою тему.
